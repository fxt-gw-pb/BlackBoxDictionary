# 医学统计与数据科学方法字典

这是一个面向医学研究、临床预测、流行病学和生物医学数据分析的 Markdown 知识库。目标不是只罗列“方法名称”，而是把每个方法整理成可直接用于学习、选型、实现和解释结果的方法卡片。

这个字典希望同时回答 4 个问题：

- 这个方法在解决什么问题
- 这个方法背后的数学形式是什么
- 这个方法在医学研究里什么时候该用、什么时候不该用
- 这个方法如何用代码实现、如何解释结果、如何可视化

## 项目定位

建议把这个项目建设成一个“方法词典 + 选型手册 + 实现备忘录”的结合体。

- 对学习来说，它帮助你把概念、公式、直觉和代码放在同一页里
- 对实战来说，它帮助你从研究问题快速映射到方法
- 对写作来说，它帮助你沉淀结果解读、图表表达和常见陷阱

适合覆盖的内容包括：

- 经典医学统计
- 回归建模与广义线性模型
- 生存分析与纵向数据分析
- 因果推断
- 树模型、聚类、降维、时间序列与高维建模
- 分类与判别方法
- 模型评估、校准、解释与可视化

## 方法卡的统一框架

建议每个方法/模型都按同一套结构编写。当前推荐用 16 个主标题（在原 12 节基础上，新增「核心思想与原理」「手把手算例」「假设诊断与稳健性」「关键术语」四节，以揭示方法本质、并对初学者友好）：

1. 方法概览（含一句话本质、直觉与类比）
2. 核心思想与原理
3. 数学形式（含推导脉络、假设及违反后果）
4. 手把手算例
5. 数据形式与输入输出
6. 适用场景
7. 实现
8. 结果如何解读
9. 假设诊断与稳健性
10. 推荐可视化
11. 优势、局限与常见坑
12. 与相近方法的区别
13. 医学研究中的典型应用
14. 关键术语
15. 相关方法
16. 参考资料

每个主标题的填写要求以 `02_模板/01-方法卡模板.md` 为准，模板是结构规范的唯一来源，README 不再重复细节。

## 编写规范

### 命名建议

- 一个方法一篇笔记
- 同一目录内建议按阅读顺序加前缀编号，文件名建议使用 `NN-中文名（英文名）.md`
- 首次出现时写全称，后文再用简称
- **aliases 必须包含全名**：每篇方法卡的 YAML `aliases` 中必须包含「文件名去掉 `NN-` 前缀后的全名」（即 `中文名（英文名）`）。全库卡片之间的双链统一写 `[[中文名（英文名）]]`，靠这个 alias 解析；漏填会导致所有指向该卡的链接失效

例如：

- `01-Logistic回归（Logistic Regression）.md`
- `02-Cox比例风险模型（Cox Proportional Hazards Model）.md`
- `03-随机森林（Random Forest）.md`

### 内容规范

- 优先写“这个方法解决什么问题”，再写“它怎么推导”
- 公式要尽量配参数解释
- 代码示例尽量给出 Python 和 R 两个版本，至少保留一个主版本
- 每篇方法卡尽量写出“适用场景”和“不适用场景”
- 每篇都保留“结果解释”和“可视化建议”这两个部分
- 每篇末尾尽量给出简洁的外部参考资料，优先使用教材、方法论文和官方文档，不把本地笔记作为正式参考文献

### 医学场景补充建议

对于医学统计方法，尤其建议额外记录：

- 结局类型：连续型、二分类、计数型、时间到事件、多分类、重复测量
- 缺失机制：MCAR、MAR、MNAR
- 是否涉及删失
- 是否需要考虑混杂、选择偏倚、类别不平衡
- 是否强调可解释性、可校准性、可外部验证

### 状态词表

`status` 字段与 `03_索引/01-方法总索引.md` 中的「状态」列使用同一套词表，两处必须保持一致：

| 状态 | 含义 |
| --- | --- |
| 待写 | 已列入计划，尚未建立文件（仅出现在索引中） |
| 草稿 | 文件已建立，内容未写完 |
| 已建 | 十二个部分内容完整，未经复核 |
| 已校 | 内容已复核定稿 |

## 维护规则（供 AI 助手与自动化工具遵循）

以下规则是硬性约定，任何人和任何 AI 助手修改本仓库时都必须遵守。规则设计为可机械执行，不需要理解全库背景。

### 新增一张方法卡

按顺序执行，缺一不可：

1. 确定分类目录：在 `01_方法词典/` 的 16 个目录中选择；归属有疑问时以 `00_项目说明/01-方法分类框架.md` 为准
2. 文件命名：`NN-中文名（英文名）.md`，`NN` 取该目录内现有最大编号 +1，括号用全角
3. 复制 `02_模板/01-方法卡模板.md` 并填写全部 frontmatter 字段：
   - `aliases` 必须包含全名（文件名去掉 `NN-` 前缀）
   - `slug` = english_name 全小写、非字母数字字符替换为 `-`，全库唯一，建立后不可更改
   - `category` 必须与所在目录名（去掉 `NN_` 前缀）一致
   - `status` 只能取：草稿 / 已建 / 已校
4. 正文保留模板全部 16 个主标题，不增删、不改序号；旧卡逐步迁移到 16 节结构
5. 双链一律写全名：`[[中文名（英文名）]]`；允许指向尚未建卡的方法（按同样格式预构造）
6. 相关方法只维护在正文第 11 节；frontmatter 不设 related_methods 字段
7. 配图放 `04_示例图像/`，文件名用英文 snake_case；卡内用相对路径 `../../04_示例图像/xxx.png` 引用；图必须能由卡内第 5 节示例代码重现；不留未被引用的图
8. 在 `03_索引/01-方法总索引.md` 增加一行，方法列写 `[[全名\|短名]]`，状态列与 frontmatter `status` 一致
9. 运行 `python3 scripts/check.py`，0 错误才算完成

### 主条目与变体条目

同族方法（如 随机森林 与 随机森林回归、K近邻算法 与 K近邻回归）：

- 主条目完整讲原理；变体条目只写与主条目的差异，开头链接主条目
- 变体条目与主条目放在同一分类目录
- 索引备注列为主条目标注「主条目」

### 修改与删除

- 不重命名已有文件和目录：alias、slug、双链和外部引用都依赖现有名字
- 修改卡片状态时，frontmatter 与索引两处必须同步
- 不新建顶层目录；不把 `~/Datasets` 下的数据集提交进 git
- 不为了让检查通过而修改 `scripts/check.py` 的判定逻辑
- 任何批量修改后运行 `python3 scripts/check.py` 复查，并在提交信息中说明改动范围

## 网站（黑盒词典）

方法卡可一键生成为静态网站「黑盒词典」，供桌面深度阅读与检索：

- 生成器：`python3 scripts/build_site.py`，读取 `01_方法词典/` 的全部方法卡，输出到 `docs/`（Hero 首页、目录页、各分类页、逐卡详情页）
- 视觉沿用 `黑盒词典网站设计/` 的 Claude Design 稿：纸白/墨黑 + 朱砂强调，Noto Serif SC / Spectral / IBM Plex Mono 三字体，KaTeX 公式、Python/R 代码分栏、粘性目次、深色模式、全局检索
- 部署：`docs/` 已含 `.nojekyll`；在 GitHub 仓库 Settings → Pages 中选择 `main` 分支 `/docs` 目录即可发布
- 任何方法卡改动后，依次重跑 `python3 scripts/check.py` 与 `python3 scripts/build_site.py`，保持内容、索引与站点三者同步
