# 数据资产指南

本文从 README 拆出，是案例数据集的选用手册。数据集的下载方式与本机存放位置见 [代码与数据集.md](代码与数据集.md)。

`05_数据分析案例` 的主要价值不在于直接复用整套分析代码，而在于把其中的数据集沉淀成这个知识库的“可复用演示数据资产”。

后续在新增方法卡时，优先考虑复用这些数据集来做：

- 数据表结构示例
- 变量类型示例
- 清洗前后对比示例
- 可视化图像示例
- Python / R 最小可运行脚本示例

## 复用原则

- `05_数据分析案例/code` 中的脚本只作为理解数据集与分析思路的参考，不作为本知识库的方法实现规范。
- 本知识库新增方法卡时，应优先重写“最小、清晰、可解释”的演示代码，而不是直接复制案例脚本。
- 复用重点放在“数据形态演示”和“图表演示”，而不是完整复刻原项目。
- 若案例数据不适合某方法的最小示例，可以补充使用 Python 和 R 的内置数据集。
- 如果案例标题与真实数据结构不完全一致，应以数据文件本身的结构为准。
- 如果案例文档中的相对路径、清洗步骤或建模流程与这里的整理方式不完全一致，应以当前知识库中的数据文件和方法卡写作目标为准。

## 重点数据特点

### 1. 时间序列与面板型数据

- `GlobalTemperatures.csv` 适合演示单变量时间趋势、多变量同步趋势、缺失值插补、滚动窗口平滑。
- `all_stocks_5yr.csv` 适合演示“`date + group + value`”这种长表结构，尤其适合按股票代码分组的折线图、收益率计算、波动分析。
- 这两类数据非常适合补充到 `描述统计`、`可视化`、`时间序列`、`非参数平滑` 等条目。

### 2. 宽表监督学习数据

- 房价、客户流失、乳腺癌、信用卡欺诈都属于“每行一个样本、每列一个变量”的经典宽表。
- 其中房价数据是**连续结局回归**的好例子，流失/乳腺癌/欺诈属于**二分类**例子。
- 这类数据非常适合演示：
  - 描述性统计表
  - 缺失值统计表
  - 数值/分类变量混合清洗
  - 训练集/测试集拆分
  - 回归、分类、正则化、模型评估

### 3. 关系型与半结构化多表数据

- 电影数据集不是单表，而是多个 CSV 组合而成，且 `genres`、`keywords`、`cast`、`crew` 等字段包含类 JSON 字符串。
- 它非常适合演示：
  - 多表合并
  - 一对多关系
  - 嵌套字符串解析
  - 高基数分类变量汇总
  - 大文件抽样分析
- 但它不适合作为“最小方法示例”的默认首选，因为清洗成本明显更高。

### 4. 类别不平衡与异常检测数据

- `creditcard.csv` 是非常典型的极端不平衡数据：`Class=1` 只有 492 条。
- 这类数据很适合未来在以下方法卡中做演示：
  - 逻辑回归在不平衡数据下的局限
  - Precision / Recall / F1 / PR 曲线
  - 重采样、阈值调整、异常检测
- 但不适合直接作为“箱线图演示”的首选，因为全量作图容易被多数类淹没，通常需要抽样。

### 5. 文本型数据

- `spam.csv` 清洗前有 5 列，但真正核心列只有标签列和短信文本列，其余 3 列本质上是空白占位列。
- 它非常适合演示：
  - 文本清洗
  - 字符串长度分布
  - 词频统计
  - TF-IDF
  - 朴素贝叶斯 / 逻辑回归文本分类
- 与医学统计主线不同，但对“数据类型扩展”很有价值。

### 6. 纵向与生存数据

- `Framingham_data.csv` 是这批数据资产中最适合医学统计主线演示的资源之一。
- 它同时支持：
  - 基线横断面分析
  - 重复测量纵向分析
  - 生存结局分析
  - 风险因素与效应修饰演示
- 未来如果要写混合效应模型、GEE、Kaplan-Meier、Cox 比例风险模型，这个数据集应该作为优先演示数据。

### 7. 标题与真实数据结构不完全一致的案例

- `007-基础的癌症生存率数据探索` 按当前数据文件内容看，实际更接近**乳腺癌良恶性分类数据**，并不包含真正的生存时间和删失结局。
- 因此后续若写“生存分析”方法条目，不建议用这个数据集做主示例，应优先用 `Framingham_data.csv` 或其他真正含 `time + event` 的数据。

## 推荐优先级

如果后续目标是给“医学统计与数据科学方法字典”补最稳妥、最通用的演示数据，建议优先级如下：

1. `Framingham_data.csv`
2. `house-prices-advanced-regression-techniques/train.csv`
3. `Churn_Modelling.csv`
4. `GlobalTemperatures.csv`
5. `creditcard.csv`
6. `Mall_Customers.csv`
7. `spam.csv`

其中：

- 医学统计主线优先用 `Framingham_data.csv`
- 经典表格预测任务优先用房价 / 流失 / 乳腺癌
- 数据可视化基础演示优先用气温 / 股票 / Mall Customers
- 文本和极端不平衡场景分别优先用短信 / 信用卡欺诈

## 可补充使用的内置数据集

当案例数据不适合做最小示例时，可以补充使用这些常见内置数据集：

- Python：`seaborn.load_dataset("tips")`、`penguins`、`titanic`
- scikit-learn：`load_breast_cancer()`、`load_diabetes()`
- R：`iris`、`mtcars`、`airquality`、`survival::lung`

这些内置数据集更适合：

- 快速写最小代码示例
- 演示单一统计概念
- 避免大型真实数据集带来的清洗噪声
