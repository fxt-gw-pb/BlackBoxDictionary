## 2. 深化写作规则（决定一张卡"深不深"）

- **揭示本质 > 罗列步骤**：§2 必须讲清「不用它会撞上什么根本困难 + 用什么关键洞察绕过」。这是最重要的一节。
- **先直觉后公式**：每节先给白话/类比，再上公式；公式后紧跟一句「这个式子在说……」。
- **手把手算例（§4）不可省**：3–8 个观测，逐步代入公式，展示中间量，最后从数字读出结论并翻译成人话。范例见已完成卡：
  - 相关三卡（Pearson/Spearman/Kendall）**共用同一份 5 点数据**得 r=0.90 / ρ=0.90 / τ=0.80，直观对照。
  - ANOVA 卡手算 F=12，TukeyHSD 卡**沿用同一数据**接着算 HSD=2.51 —— 例子前后咬合是加分项。
  - Poisson 手算 IRR=2.0=exp(0.69)；负二项由方差 20/均值 5 解出 α=0.6；Quasi 手算 φ=3、SE×√3 后结论翻转。
- **难以手算的方法**（如 RCS 的样条基）：§4 改走"工作流"式算例（选结点→数自由度→非线性检验→读效应曲线），并诚实说明为何不逐项手推。
- **§14 术语表**：把本卡出现的关键术语各用一句话中英对照解释。
- 医学语境：标注结局类型、缺失机制、删失、混杂、类别不平衡等。

## 3. 硬性技术约定（违反会破坏站点或检查）

1. **frontmatter 的 `slug` 与 `aliases` 一字不改**：改写时必须原样保留（slug 是网站 URL、双链靠 aliases 全名解析）。改写前先 `Read` 原卡取这两项。
2. **保留原卡的配图引用**：否则 `check.py` 报「孤儿图，未被任何方法卡引用」错误。把原来的 `![](../../04_示例图像/xxx.png)` 迁到 **§10 推荐可视化**（可加一个 `### 10.1 图像示例` 小节）。改写前先 grep 原卡有无配图。
3. **LaTeX 规范**（`build_site.py` 会转义，但仍应写干净）：
   - **不要用裸 `<` `>`**（会被浏览器当 HTML 标签破坏 DOM）；用 `\lt \gt \le \ge` 或中文"小于/大于"。
   - 花括号必须配平；`\frac` 分母要带 `{}`。
   - KaTeX 支持 `\binom \dbinom \begin{aligned} \begin{smallmatrix}` 等；`\begin{align}`（无 aligned）、`\bm`、`\argmin` 不支持。
4. **代码分栏**：§7 实现必须有 `### 7.1 Python` 和 `### 7.2 R` 两个子标题（`build_site.py` 靠"标题含'实现' + 存在 `### x.y`"来渲染 Python/R 分栏）。
5. **双链写全名** `[[中文名（英文名）]]`；相关的未建卡也可链（允许，显示为提示）。
6. **不改文件名/编号/目录**；一次只改正文，frontmatter 只在必要时（如 status）动。

## 4. 每张卡的工作流

1. `Read` 原卡（`limit: 1` 即可登记，但要先用 grep/sed 抓 `slug`、`aliases`、配图路径）。
2. `Write` 全文改写为 16 节（保留 slug/aliases/配图）。
3. 一批（4–8 张）写完后：`python3 scripts/check.py` → 必须 **0 错误 0 提示**。
4. `python3 scripts/build_site.py` 重新生成 `docs/`。
5. **KaTeX 审计**（强烈建议）：用 Preview 打开任一详情页加载 KaTeX，再 fetch 本批各页、剥掉 `<pre><code>`、对每个 `$$…$$`/`$…$` 跑 `katex.renderToString({throwOnError:true})`，确认 0 失败、各页 16 个 TOC 项、2 个代码分栏。（本会话用的审计 JS 片段可复用；核心是"逐条公式送进 KaTeX 验证"。）
6. `git add -A && git commit`（提交信息说明本批改了哪些、手算了什么）。
7. 推送（见 §5）。
8. 验证部署成功（见 §5）。
