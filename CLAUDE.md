# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是一个 Obsidian 中文知识库（非代码项目）：「医学统计与数据科学方法字典」。核心产出是 Markdown 方法卡片，把每个统计/机器学习方法的概念、公式、适用场景、Python/R 实现和结果解释放在同一页里。没有构建、测试或 lint 命令；唯一必须运行的程序是一致性检查器 `python3 scripts/check.py`（任何内容改动后都要运行，0 错误才算完成）。

本库是 git 仓库，远程为 `git@github.com:fxt-gw-pb/BlackBoxDictionary.git`（与 GitHub 交互一律用 SSH 地址）。大型原始数据集不入库：实际存放在 `~/Datasets/黑箱字典/dataset/`，库内 `05_数据分析案例/code/dataset` 是指向它的符号链接，已被 .gitignore 排除。

**维护规则的唯一权威来源是 README「维护规则（供 AI 助手与自动化工具遵循）」一节**，涵盖新增方法卡的完整步骤、命名/alias/slug/status 约定、主条目与变体条目分工、禁止事项。本文件只做导航，不重复规则细节。

**正在进行的长期任务：把全部方法卡升级到 16 节深化标准。** 接手前必读 `00_项目说明/07-方法卡深化规则与进度.md`——内含深化写作规则、硬性技术约定（保留 slug/aliases/配图、LaTeX 禁裸 `<>`、代码分栏靠 `### 7.x`）、每卡工作流、校验/推送/部署命令（推送须走 SSH-over-443）、以及逐类进度快照与下一步。

## 目录结构与职责

- `README.md` — 项目地图 + 编写规范 + **AI 维护规则（权威）**
- `scripts/check.py` — 一致性检查：双链解析、全名 alias、slug 唯一性、status 词表、图像引用、索引同步
- `scripts/build_site.py` — 静态站点生成器：读 `01_方法词典/` 全部方法卡，按 `黑盒词典网站设计/` 的视觉稿输出到 `docs/`（GitHub Pages）；改动方法卡后需重跑
- `docs/` — 生成的「黑盒词典」网站，勿手改（改内容后重跑生成器）；`黑盒词典网站设计/` 为 Claude Design 视觉稿源
- `00_项目说明/` — 分类框架详解、首批方法清单、生物统计课程映射、网站设计 Brief；`07-方法卡深化规则与进度.md` 为 16 节深化任务的**交接文档**（接手深化工作先读它）
- `01_方法词典/` — 主体内容，16 个编号分类目录（01_研究设计与数据理解 … 16_分类与判别方法），一个方法一张卡
- `02_模板/01-方法卡模板.md` — 方法卡结构的唯一规范（16 个主标题 + YAML frontmatter，含 slug 与全名 alias 的填写说明；16 节含「核心思想/手把手算例/假设诊断/关键术语」等揭示本质、面向初学者的部分）
- `03_索引/01-方法总索引.md` — 进度跟踪 + 可点击导航；方法列为 `[[全名\|短名]]` 链接，状态列与卡片 frontmatter 同步
- `04_示例图像/` — 方法卡配图，英文 snake_case 命名，卡内以 `../../04_示例图像/xxx.png` 相对路径引用
- `05_数据分析案例/` — `00-数据资产指南.md`（数据集选用手册）、案例笔记 001-010、`代码与数据集.md`（下载与本机位置）、`code/`（参考脚本）、`Framingham研究/`（含库内保留的 512K Framingham 数据）

## 关键约定速查

- 双链一律写全名 `[[中文名（英文名）]]`，靠每张卡 aliases 中的全名解析；允许指向尚未建卡的方法
- `status` 词表：待写（仅索引）/ 草稿 / 已建 / 已校；frontmatter 与索引两处必须同步
- `slug` 用于网站 URL：english_name 全小写、非字母数字替换为 `-`，全库唯一，建立后不可更改
- frontmatter 不设 related_methods；相关方法只维护在正文第 11 节
- 不重命名已有文件/目录，不新建顶层目录，不把数据集提交进 git

## 演示数据的选用规则

方法卡代码示例的数据选用详见 `05_数据分析案例/00-数据资产指南.md`，要点：

- 医学统计主线（生存分析、纵向数据、混合效应、GEE 等）优先用 `Framingham_data.csv`（含重复测量、删失、真实 time + event 结局）
- 回归/分类演示优先用房价（`003`）、客户流失（`004`）、乳腺癌（`007`）数据
- 注意：案例 007 标题写「生存率」但实际是乳腺癌良恶性分类数据，不含生存时间，不要用它演示生存分析
- 案例数据不合适时改用内置数据集（seaborn tips/penguins、sklearn load_breast_cancer、R 的 survival::lung 等）
- `05_数据分析案例/code/` 中的脚本仅作理解数据集的参考——方法卡的代码一律重写为「最小、清晰、可解释」的版本

## 内容规范要点

- 写作顺序：先「解决什么问题」再「怎么推导」；公式必须配参数解释；代码给 Python 和 R 两个版本
- 每篇方法卡都要有「适用/不适用场景」「结果解释」「推荐可视化」；配图须能由卡内第 5 节示例代码重现
- 医学场景需标注：结局类型（连续/二分类/计数/时间到事件/重复测量）、缺失机制、是否删失、是否有混杂或类别不平衡
- 分类按「具体方法家族」组织（见 `00_项目说明/01-方法分类框架.md`）；判别类方法（KNN 家族、朴素贝叶斯、LDA/QDA）归入 `16_分类与判别方法`
